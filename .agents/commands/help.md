# Command: /help

Uses:
- Command router summary.

Example:

```txt
/help
```

Response should show:
- `/bug`, `/debug` - debugging workflow.
- `/review` - code review workflow.
- `/fix <task>` - structured fix workflow.
- `/ui`, `/frontend` - frontend UI workflow.
- `/security` - security audit workflow.
- `/deploy`, `/devops` - deployment workflow.
- `/docs`, `/research` - current docs and source verification.
- `/refactor` - behavior-preserving cleanup.
- `/test` - test strategy and implementation.
- `/use <skill1> <skill2>` - combine skills.
- `/mode <mode>` - activate session-wide behavior.
- `/mode reset` - clear active modes.
