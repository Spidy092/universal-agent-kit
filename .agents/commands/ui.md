# Command: /ui

Aliases:
- `/ui`
- `/frontend`

Uses:
- `frontend-ui`
- Add `testing` for interaction or regression coverage.
- Add `security-audit` when UI changes affect auth, payments, tokens, or permissions.

Examples:

```txt
/ui fix mobile navbar
```

```txt
/frontend improve login form error states
```

Workflow:
1. Inspect the existing UI patterns.
2. Make responsive, accessible changes.
3. Verify in browser when possible.
4. Provide a screenshot or artifact when possible.
