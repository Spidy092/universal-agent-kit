# Frontend UI Skill

## When to Use

Use when building UI components, fixing layout, making pages responsive, improving UX, adding animation, integrating assets/SVG/layers, or improving accessibility.

## Inputs to Gather

- target component/page
- framework
- design system
- asset paths
- breakpoints
- states required
- browser/mobile verification path

## Workflow

1. Inspect existing UI patterns.
2. Reuse existing components first.
3. Implement desktop layout, mobile layout, loading state, empty state, error state, disabled state, accessible labels, and keyboard behavior.
4. Keep animation useful and not heavy.
5. Verify with viewport sizes.
6. Mention design assumptions.
