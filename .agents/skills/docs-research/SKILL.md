# Docs Research Skill

## When to Use

Use when API behavior may have changed, library/framework details matter, the user asks "are you sure," tool/CLI behavior is unclear, implementation depends on current docs, or before saying "not possible."

## Goal Tracking

Before starting, define:
- **Research Goal**: What specific information do we need?
- **Reliability**: How certain do we need to be?
- **Done Criteria**: How will we know we have the answer?

## Inputs to Gather

- exact library/tool/framework
- version if known
- exact question
- local code path affected
- desired output: answer, implementation plan, workaround, source-code modification, or prototype/hack path

## Workflow

### Phase 1: Question Definition (3 min)
- [ ] Define the exact question
- [ ] Identify what would constitute a good answer
- [ ] Plan research sources

### Phase 2: Source Research (10-15 min)
- [ ] Check official docs first
- [ ] Check official GitHub repo/source
- [ ] Check CLI help if it is a CLI/tool
- [ ] Check issues/discussions if behavior is unclear
- [ ] Check community resources if needed

### Phase 3: Verification (5 min)
- [ ] Separate confirmed facts, assumptions, and guesses
- [ ] Verify with multiple sources
- [ ] Check for version-specific behavior
- [ ] Document confidence level

### Phase 4: Presentation (5 min)
- [ ] Present findings clearly
- [ ] Provide all viable routes
- [ ] Document risks and tradeoffs
- [ ] Give next steps

## Research Sources

### Primary Sources (Most Reliable)
- Official documentation
- Official GitHub repository
- Official CLI help
- Official blog/changelog

### Secondary Sources (Verify)
- Stack Overflow answers
- Community forums
- Blog posts
- Tutorial sites

### Tertiary Sources (Use with Caution)
- AI-generated content
- Unofficial documentation
- Forum answers without verification

## Verification Methods

### Code Verification
- Read source code directly
- Run test cases
- Check implementation details

### Documentation Verification
- Check multiple documentation pages
- Verify version compatibility
- Check for deprecation notices

### Community Verification
- Check multiple Stack Overflow answers
- Verify with official issues
- Check community forums

## Output Format

## Confirmed facts
## Assumptions
## Recommended route
## Official method
## Workaround
## Source-code modification route
## Prototype/hack route
## Risks
